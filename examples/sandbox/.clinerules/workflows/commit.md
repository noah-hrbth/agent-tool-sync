
1. Run `git diff --staged` and `git status` to understand pending changes
2. Infer the commit type: `feat` / `fix` / `chore` / `docs` / `refactor` / `test`
3. Write a one-line conventional commit message: `<type>(<scope>): <description>`
4. Stage all relevant files and commit — never commit `.env`, secrets, or build artifacts
5. Print the commit hash when done
